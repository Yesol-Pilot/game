# 💰 결제 연동 및 사업자 등록 가이드

## 1. 현재 상태: 테스트 모드 (개발용)
- **사업자 등록증**: ❌ **필요 없음**
- **기능**: 결제창이 뜨고, 카드 선택 및 인증 과정을 **시뮬레이션** 할 수 있습니다.
- **실제 돈**: 빠져나가지 않습니다. (매출 발생 X)
- **용도**: 게임의 상점 기능이 정상적으로 돌아가는지 확인하는 개발 단계용입니다.

## 2. 라이브 모드 (실제 돈 받기)
- **사업자 등록증**: ⭕ **필수** (한국 PG사 계약 시)
- **PortOne(아임포트)**: 한국 결제 대행사(KG이니시스, 토스페이먼츠 등)와 계약하려면 **사업자 등록증**과 **통신판매업 신고증**이 반드시 필요합니다.
- **예외 (개인 개발자)**:
    - **구글 플레이 인앱 결제 (IAP)**: 구글 콘솔에서 '개인 개발자'로 등록하면 사업자 없이도 수익 창출이 가능합니다.
    - **중요**: 현재의 '웹 결제 SDK(PortOne)'는 사용할 수 없으며, 안드로이드 네이티브(Bubblewrap/TWA) 빌드 후 **Google Play Billing Library**를 연동해야 합니다.

## 3. 솔루션: 개인 개발자를 위한 현실적인 대안
사업자 등록 없이 당장 수익을 내고 싶다면 아래 방법뿐입니다.

### A. 안드로이드 앱 출시 (Google Play)
1.  이 웹 게임을 TWA(Bubblewrap)로 패키징하여 Play Store에 올립니다.
2.  PortOne 대신 **Play Store 결제 라이브러리**를 연동합니다. (구글 수수료 15~30%)
3.  개인 통장으로 정산받을 수 있습니다.

### B. 후원 링크 (Donation) - *추천*
웹 게임 상태를 유지하면서 수익을 얻으려면 **후원 플랫폼**을 연결하는 것이 가장 빠릅니다.
- **Patreon, Ko-fi, Buy Me a Coffee** 등의 링크를 상점에 '후원하기' 버튼으로 연결합니다.
- 기술적으로 가장 간단하고 법적 제약이 적습니다.

## 4. 코드 설정 방법 (PaymentManager.js)
`js/managers/PaymentManager.js` 파일을 열어보세요.
```javascript
this.IS_TEST_MODE = true; // false로 바꾸면 실제 결제 모드
this.LIVE_STORE_ID = "..." // 사업자 키 입력
```
사업자 등록 후 키를 발급받으면 위 변수만 수정하면 됩니다.
